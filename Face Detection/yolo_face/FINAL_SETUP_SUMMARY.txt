================================================================================
                            CONFIGURATION
================================================================================

Confidence Threshold (--conf):
  0.3-0.4  = More detections, more false positives (too sensitive)
  0.5-0.6  = Balanced (RECOMMENDED)
  0.7-0.8  = Fewer detections, very strict

Output Port (--serve-port):
  Default 8004, change to any available port

Input Source (--source):
  "https://..." = MJPEG stream (with --insecure for self-signed cert)
  "usb0"        = USB camera
  "video.mp4"   = Video file
  "image.jpg"   = Image file

Display Mode:
  --no-display = Run headless (no window)
  (remove flag) = Show cv2.imshow() if DISPLAY available

HTTP Broadcasting:
  --serve --serve-port 8004 = Enable HTTP MJPEG output

================================================================================
                         TROUBLESHOOTING
================================================================================

"0 faces detected" → Face not in stream content
  Solution: Test with known face image
  $ python yolodesk --model models/yolov8n-face.pt detect --source test_image.jpg

"Cannot connect to MJPEG" → Network issue
  Solution: Verify stream is accessible
  $ curl -k https://192.168.1.111:8002/mjpeg

"Port already in use" → Another process using port
  Solution: Kill existing process or use different port
  $ pkill -f "yolodesk.*8004"
  $ python yolodesk ... --serve-port 8005

"ultralytics not available" → Expected, using Haar Cascade
  This is normal - yolodesk falls back to available backend
